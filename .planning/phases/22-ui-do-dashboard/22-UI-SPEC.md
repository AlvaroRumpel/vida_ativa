## Spacing Scale

Declared values — matches existing `AppSpacing` tokens (source: `lib/core/theme/app_spacing.dart`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Card padding within grid, badge gaps, color strip width |
| sm | 8px | Metric label↔value gap in card, inline element gaps |
| md | 16px | Default horizontal padding on dashboard tab, chart margins |
| lg | 24px | Section padding, form horizontal padding, bottom sheet padding |
| xl | 32px | Layout gaps between major sections |

Exceptions:
- KPI grid: `GridView` with `crossAxisSpacing: 8px` (sm)
- Chart container margins: top 8px, left/right 8px (sm)
- Period toggle container: top 16px, left/right 16px (md)

## Typography

Source: `lib/core/theme/app_theme.dart` — `GoogleFonts.nunitoTextTheme()` applied globally.
All font sizes and weights below match patterns observed in existing UI files.

Exactly 6 sizes declared:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Timestamp | 11px | 400 (regular) | 1.4 |
| Card label (KPI) | 12px | 400 (regular) | 1.4 |
| Body | 14px | 400 (regular) | 1.5 |
| Section title | 16px | 600 (semibold) via `textTheme.titleMedium` | 1.3 |
| Heading (chart title) | 18px | 600 (semibold) | 1.2 |
| KPI card value | 24px | 700 (bold) — Material 3 `displaySmall` approximation | 1.2 |

Notes:
- Timestamp text "Última atualização": 11px / w400 (de-emphasized, gray color)
- KPI card label (e.g., "Taxa de Ocupação"): 12px / w400 — smaller than body to fit compact card
- Period toggle label: 14px / w600 (matches existing segment button text)
- Section title: 16px / w600
- Chart title: 18px / w600 above each chart (BarChart, Heatmap, PieChart, DoughnutChart)
- KPI card value (metric number): 24px / w700 — bold prominence
- Chart axis labels: 12px / w400 (fi_chart defaults)

## Component Inventory

### New: `DashboardTab` widget

- Widget type: `StatefulWidget` (manages local `selectedPeriod` state)
- Parent integration: inserted as **Tab(text: 'Dashboard')** at index 0 in `AdminScreen.TabBar`
- Structure: `Column` → SegmentedButton + Timestamp + Divider + Expanded(SingleChildScrollView(...))
- State tracking: internal `String _selectedPeriod = 'week'` (values: 'week', 'month', 'year')
- Scaffold index update: existing `_reservasTabIndex = 2` becomes `_reservasTabIndex = 3` in AdminScreen

### New: Period Toggle — `SegmentedButton<String>`

| Property | Value |
|----------|-------|
| Type | `SegmentedButton<String>` |
| Position | Top of tab, outside scroll, fixed below AppBar |
| Segments | 3 buttons: "Semana", "Mês", "Ano" |
| Default value | 'week' |
| Padding | `EdgeInsets.fromLTRB(16, 16, 16, 0)` (md spacing) |
| onSelectionChanged | Updates `_selectedPeriod`, triggers `BlocBuilder` rebuild |
| Style | `ButtonStyle` inherits from `SegmentedButton.styleFrom(...)` with Material 3 defaults |

### New: Timestamp Line — `Text` (Last Update)

| Property | Value |
|----------|-------|
| Copy | `'Última atualização: HH:MM'` or `'Última atualização: --'` if null |
| Font style | 11px / w400, `Colors.grey[600]` |
| Position | Below SegmentedButton, right-aligned |
| Padding | `EdgeInsets.symmetric(horizontal: 16, vertical: 4)` |
| Data source | `DashboardData.updatedAt` for selected period |
| Formatter | `intl.DateFormat('HH:mm').format(updatedAt)` if not null, else `'--'` |

### New: KPI Grid — `GridView.count`

| Property | Value |
|----------|-------|
| Layout | 2 columns (responsive: 1 col on phone width < 600) |
| Crossaxis spacing | 8px (sm) |
| Main axis spacing | 8px (sm) |
| Padding | `EdgeInsets.all(16)` (md) |
| Item count | 5 KPI cards (occupancy, revenue, avg ticket, conversion, no-show) |

**KPI Card (single item):**

| Property | Value |
|----------|-------|
| Widget type | `Card` → `Padding` → `Column` |
| Padding | `EdgeInsets.all(12)` (sm + xs) |
| Label | 12px / w400 gray text (e.g., "Taxa de Ocupação") |
| Value | 24px / w700 primaryGreen text (e.g., "85%") |
| Unit | 12px / w400 gray text below value (e.g., "de ocupação") — optional |
| Shadow | Material 3 default (elevation 1) |
| On error (null metric) | Display `'--'` in place of value; no spinner |

| Property | Value |

### New: Heatmap (Hour×Day) — DASH-06

### New: PieChart (Status Breakdown) — DASH-07

| Property | Value |
|----------|-------|
| Chart type | `PieChart` (fl_chart) |
| Title | "Distribuição de Reservas por Status" (18px / w600) |
| Container | `Card` → `Padding(16)` → `AspectRatio(16/9)` → `PieChart` |
| Sections | 4 pie slices: confirmed, cancelled, pending, expired (from `DashboardData` fields) |
| Colors | Material 3 colorScheme palette (auto-assigned per slice) |
| Legend | Below chart with labels (confirmed, cancelled, pending, expired) |
| Center text | Optional: total count or percentage |
| On null data | Display `'--'` in place of chart |

Data mapping:
- Confirmed: `confirmedBookings`
- Cancelled: `cancelledBookings`
- Pending: `pendingBookings`
- Expired: calculated as `totalBookings - confirmedBookings - cancelledBookings - pendingBookings`

### New: DoughnutChart (Revenue by Sport) — DASH-08

|----------|-------|

| Sections | N slices per `revenueBySport` list; if null/empty, display fallback |
| Colors | Material 3 colorScheme palette (auto-assigned per sport) |

Data mapping:
- Per entry in `revenueBySport`: sport name → revenue amount
- Null sport entry rendered as "Não informado"

## Interaction States

### Loading State (`DashboardLoading`)

| Element | Visual |
|---------|--------|
| Entire tab | `Center(child: CircularProgressIndicator())` |
| Period toggle | Hidden (behind loading spinner) |

### Loaded State (`DashboardLoaded`)

| Element | Visual |
|---------|--------|
| Period toggle | Visible, user can select week/month/year |
| Timestamp | Visible below toggle (shows `HH:MM` or `--`) |
| KPI grid | Visible, metrics show value or `'--'` if null |
| BarChart (revenue) | Visible, bars rendered or `'--'` if all values zero |
| Heatmap | Visible, gradient rendered or message if empty |
| PieChart (status) | Visible, slices rendered or `'--'` |
| DoughnutChart (sport) | Visible with data or fallback message |
| Scroll | User can scroll down to see all charts |

### Error State (`DashboardError`)

| Element | Visual |
|---------|--------|
| Entire tab | `SnackBar` with error message + "Tentar Novamente" action button |
| Trigger | Calls `context.read<DashboardCubit>().retry()` or manual Cubit reconstruction |

### Empty/Null Data Handling

| Metric | On Null | Display |
|--------|---------|---------|
| Occupancy rate | null | `'--'` |
| Ticket médio | null | `'--'` |
| Conversion rate | null | `'--'` |
| No-show rate | null | `'--'` |
| Revenue (all bars zero) | 0.0 | BarChart shows zero-height bars or `'--'` |
| revenueBySport | null or empty list | Card with message: "Nenhum dado de esporte ainda" |
| topClients | null | Metric card shows `'--'` (if included; currently not in DASH-05..08) |
