## Spacing Scale

Declared values ‚Äî matches existing `AppSpacing` tokens (source: `lib/core/theme/app_spacing.dart`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, color strip width |
| sm | 8px | Compact list item vertical padding, inline element gaps |
| md | 16px | Default horizontal padding on list screens, form field spacing |
| lg | 24px | Section padding, form horizontal padding, bottom sheet padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | not defined in AppSpacing ‚Äî avoid; use xl (32px) instead |
| 3xl | 64px | not defined in AppSpacing ‚Äî avoid |

Exceptions:
- Touch targets for `IconButton` (delete per sport item): minimum 48x48px ‚Äî Flutter Material default; no override needed
- `ReorderableListView` drag handles: 44px row height minimum ‚Äî use `ListTile` default (56px)
- Dropdown field height: matches existing `TextField` at `OutlineInputBorder` default (~56px)

## Color

Source: `lib/core/theme/app_theme.dart` ‚Äî ColorScheme.fromSeed with Material 3.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `Color(0xFFFDFAF5)` | Background surface, scaffold background, sheet background |
| Secondary (30%) | Material 3 card surface (auto from seed) | Cards (`AdminBookingCard`), `SettingsTab` form background |
| Accent (10%) | `Color(0xFF7B5D0A)` (primaryGreen) | Sport chip border, `FilledButton` background, `Icons.check_circle` confirmation |
| Destructive | `Colors.red` | Delete sport `IconButton` in `ReorderableListView` |

Accent reserved for:
1. `DropdownButtonFormField` focus border (via `ColorScheme.primary`)
2. Sport chip border color when `sport != null`
3. `FilledButton` "Salvar Esportes" in SettingsTab sports section
4. `Icons.check_circle` when sport config stream confirms save

Sport chip color palette ‚Äî deterministic hash algorithm (source: CONTEXT.md D-10):
- Algorithm: `sportName.hashCode % colors.length` ‚Üí index into fixed list
- 8 distinct Material colors (background / foreground pairs):

| Index | Background | Foreground |
|-------|-----------|------------|
| 0 | `Color(0xFFE3F2FD)` | `Color(0xFF1565C0)` ‚Äî Blue |
| 1 | `Color(0xFFE8F5E9)` | `Color(0xFF2E7D32)` ‚Äî Green |
| 2 | `Color(0xFFFFF3E0)` | `Color(0xFFE65100)` ‚Äî Orange |
| 3 | `Color(0xFFF3E5F5)` | `Color(0xFF6A1B9A)` ‚Äî Purple |
| 4 | `Color(0xFFFCE4EC)` | `Color(0xFFC62828)` ‚Äî Pink/Red |
| 5 | `Color(0xFFE0F7FA)` | `Color(0xFF00695C)` ‚Äî Teal |
| 6 | `Color(0xFFF9FBE7)` | `Color(0xFF558B2F)` ‚Äî Lime |
| 7 | `Color(0xFFFFF8E1)` | `Color(0xFFF57F17)` ‚Äî Amber |

Chip sizing: `padding: EdgeInsets.symmetric(horizontal: 8, vertical: 2)` ‚Äî matches status chip in `AdminBookingCard` line 135.

## Component Inventory

### New: `SportChip` widget
- Widget type: stateless `Container` with `BoxDecoration` ‚Äî same pattern as status chip in `AdminBookingCard`
- Inputs: `String sportName` (non-null, caller checks nullability before rendering)
- Rendering: color from hash palette above; label text = `sportName`; font: 12px / w600
- Border radius: `BorderRadius.circular(12)` ‚Äî matches status chip
- Padding: `EdgeInsets.symmetric(horizontal: 8, vertical: 2)`
- Null guard: caller (AdminBookingCard, AdminBookingDetailSheet) renders chip only when `booking.sport != null`

### Modified: `BookingConfirmationSheet`
- New field: `DropdownButtonFormField<String?>` inserted after `_participantsController` TextField (after line 350 of existing file), before error message and action buttons
- Widget: `DropdownButtonFormField` ‚Äî native Material, consistent with other fields (CONTEXT.md D-01)
- Label: "Esporte (opcional)" (CONTEXT.md D-02)
- Empty state: hide entire dropdown when `SportConfigCubit` state has empty list (CONTEXT.md D-04)
- Border: `OutlineInputBorder(borderRadius: BorderRadius.all(Radius.circular(12)))` ‚Äî matches `_participantsController` field
- State: internal `String? _selectedSport` field; passes value to `bookSlot` call

### Modified: `AdminBookingCard`
- Add `SportChip` below participants row (when `booking.sport != null`)
- Spacing: `SizedBox(height: 4)` before chip row ‚Äî matches participant row gap

### Modified: `AdminBookingDetailSheet`
- Add `_infoRow(Icons.sports, booking.sport!)` after participants row (when `booking.sport != null`)
- Icon: `Icons.sports` (Material Icons built-in)

### Modified: `SettingsTab`
- New section "Esportes" appended after existing Mercado Pago section
- Structure: `Divider` ‚Üí `Text('Esportes', style: textTheme.titleMedium)` ‚Üí `BlocBuilder<SportConfigCubit, SportConfigState>` ‚Üí sport list UI
- Sport list: `ReorderableListView` with `ListTile` per sport (CONTEXT.md D-06)
- Each `ListTile`: trailing `IconButton(icon: Icon(Icons.delete_outline), color: Colors.red)` for remove (CONTEXT.md D-08)
- Add sport: `Row` with `TextField` (inline, `Expanded`) + `IconButton(icon: Icon(Icons.add), color: primaryGreen)` (CONTEXT.md D-07)
- Save button: `FilledButton` with `backgroundColor: AppTheme.primaryGreen`, label "Salvar Esportes"

## Interaction States

### `DropdownButtonFormField` (booking form)
| State | Visual |
|-------|--------|
| SportConfigCubit loading | Dropdown hidden (not rendered) |
| SportConfigCubit loaded, list empty | Dropdown hidden (not rendered) |
| SportConfigCubit loaded, list has items | Dropdown visible, default value null ("Nenhum") |
| User selects sport | Dropdown shows sport name, `_selectedSport` updated |
| Form submitted | `_selectedSport` passed to `bookSlot` (nullable) |

### `ReorderableListView` (SettingsTab sports section)
| State | Visual |
|-------|--------|
| Loading | `CircularProgressIndicator` centered |
| Loaded, empty | Empty state text (see copywriting) |
| Loaded, has items | `ReorderableListView` with drag handles |
| User reorders | Local list updates immediately; unsaved indicator none (save required) |
| User deletes item | Item removed from local list immediately |
| User adds item | TextField cleared; item appended to local list |
| Saving | `FilledButton` shows `CircularProgressIndicator` (20x20, strokeWidth 2, color white) |
| Save success | `SnackHelper.success` toast |
| Save error | `SnackHelper.error` toast |

### `SportChip` (AdminBookingCard / AdminBookingDetailSheet)
| State | Visual |
|-------|--------|
| `booking.sport == null` | Not rendered (zero height) |
| `booking.sport != null` | Chip rendered with hash-determined color |
