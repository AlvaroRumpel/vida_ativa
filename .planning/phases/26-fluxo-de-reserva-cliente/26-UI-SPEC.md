# Phase 26 — UI Design Contract: Fluxo de Reserva (Cliente)

> Visual and interaction contract for booking confirmation sheet and My Bookings screen. Implements hero typography (Anton 88px/72px), hairline rows, and sport button patterns. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Flutter (custom AppTheme) |
| Design System | AppTheme Arena Esportivo (Phase 23 — locked) |
| Component library | Custom widgets + SlotHairlineRow (Phase 26 new), SportBtn (Phase 26 new) |
| Typography | Anton (display), Manrope (UI), JetBrains Mono (mono labels) |
| Color palette | Sand, Ink, Orange, Court, Sun, Line, LineHair, Concrete (AppTheme constants) |

## Spacing Scale

Declared values (multiples of 4, from AppTheme standard):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline decorative spacing |
| sm | 8px | Compact element spacing, hairline row padding |
| md | 16px | Default element spacing, section padding |
| lg | 24px | Section gaps, large padding |
| xl | 32px | Layout gaps, major breaks |
| 2xl | 48px | Section breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions:**
- Hairline divisor thickness: 0.5px (not a multiple of 4 — intentionally thin)
- Orange stripe width: 2px (decoration, not structural spacing)
- Hero block padding: determined in discretion section below

## Component Inventory

### NEW: SportBtn

**File:** `lib/core/widgets/sport_btn.dart`

**Purpose:** Reusable action button for Phase 26+ (Pix payment, confirm booking, save table). No icons — Anton uppercase text only.

**Variants:**

#### SportBtn.filled (orange)
- Background: `AppTheme.orange`
- Text color: `AppTheme.paper`

- Border radius: `StadiumBorder()` (pill shape)
- Min size: `Size(double.infinity, 52)` — full width, 52px height

- Background: transparent
- Text style: `AppTheme.display(size: 15, color: AppTheme.ink)` → Anton 15px uppercase

- Min size: `Size(double.infinity, 52)`
- Padding: `symmetric(horizontal: 24, vertical: 14)`

### NEW: HairlineBookingRow

**File:** `lib/features/booking/ui/hairline_booking_row.dart`

**Purpose:** Reusable row widget for MyBookings past/future bookings list. Replaces BookingCard (which becomes dead code after Phase 26).

**Layout:**
```
[ DAY_COL | TIME_COL | STATUS_PILL ]
```

**Left column (day):**
- Row(mainAxisSize: min) containing:
  - Day of month: `Text(booking.dateTime.day.toString(), style: AppTheme.display(size: 30, color: AppTheme.ink))`
  - Gap: 8px
  - Day abbreviation: `Text(dayAbbr, style: AppTheme.mono(size: 10, color: AppTheme.concrete))` — "SEG", "TER", etc.

**Middle column (time):**
- Expanded
- Horário em Anton 26px: `Text(timeFormatted, style: AppTheme.display(size: 26, color: AppTheme.ink))`
  - Example: "14:30" → formatted by `DateFormat('HH:mm')` (no seconds)

**Right column (status pill):**
- Status pill outline (no fill):
  ```dart
  Container(
    decoration: BoxDecoration(
      border: Border.all(color: statusColor, width: 1),
      borderRadius: BorderRadius.circular(16),
    ),
    padding: EdgeInsets.symmetric(horizontal: 12, vertical: 6),
    child: Text(statusLabel, style: AppTheme.mono(size: 10, color: statusColor)),
  )
  ```

**Status color mapping:**
- `confirmed` → `AppTheme.court` + label "CONFIRMADO"
- `pending_payment` → `AppTheme.orange` + label "PIX PENDENTE" (with eyebrow "AGUARDANDO PIX" above)
- `cancelled` → `AppTheme.orangeDk` + label "CANCELADO"
- `expired` → `AppTheme.concrete` + label "EXPIRADO"

**Row separator:**
- Top border: `Border(top: BorderSide(color: AppTheme.lineHair, width: 0.5))`

**Tap behavior:**
- GestureDetector wraps entire row
- If `status == 'pending_payment'` and `paymentId != null`: navigate to `PixPaymentScreen`
- Otherwise: show `ClientBookingDetailSheet(booking, bookingCubit, isFuture)`

**Spacing:**
- Horizontal padding: 16px (consistent with screen edges)
- Vertical padding: 12px (row content padding)
- Hairline between rows: 0.5px (not counted toward spacing scale)

## Interaction Patterns

### Confirmation Sheet Hero Block
- **Interaction:** Static display (no tap)
- **States:** No hover/pressed states — display only

### HairlineBookingRow
- **Interaction:** GestureDetector tap
- **States:**
  - Default: opacity 1.0
  - Pending payment + paymentId: tap → navigate to PixPaymentScreen
  - Other statuses: tap → show ClientBookingDetailSheet
  - Hover (web): subtle opacity change (optional, not mandated)

### MyBookings Hero "Próximo"
- **Interaction:** GestureDetector tap
- **States:**
  - Same as HairlineBookingRow — pending_payment routes to PixPaymentScreen, others to detail sheet

### SportBtn
- **Interaction:** onPressed callback (provided by consumer)
- **States:**
  - Default: solid background (filled) or outline (outlined)
  - Pressed: opacity 0.8 (material button default)
  - Disabled: opacity 0.38 (material button default, if enabled=false)

### Section Headers
- **Interaction:** Static (no tap)
- **States:** No interactive states

### Empty State Button
- **Interaction:** onPressed → `StatefulNavigationShell.of(context).goBranch(0)`
- **States:** SportBtn.outlined standard states
