**firestore.rules:**
1. Update `/users/{userId}` write rule to allow admin to update any user's role. Change:
   ```
   allow write: if isAuthenticated() && request.auth.uid == userId;
   ```
   To:
   ```
   allow create: if isAuthenticated() && request.auth.uid == userId;
   allow update: if isAuthenticated() &&
     (request.auth.uid == userId || isAdmin());
   allow delete: if false;
   ```
   This splits write into create/update/delete:
   - `create`: only the user themselves (registration)
   - `update`: the user themselves OR an admin (for promotion)
   - `delete`: never (no user deletion)

2. Also update `/users/{userId}` read rule to allow any admin to read ALL user docs (needed for user search). Current rule already allows `isAdmin()` to read any user — confirmed. But we also need to allow admin to LIST/query the users collection, which the current rule already supports since `isAdmin()` matches any userId.

**auth_cubit.dart:**
3. Add `promoteUser` method to AuthCubit:
   ```dart
   Future<void> promoteUser(String uid) async {
     await _firestore.collection('users').doc(uid).update({
       'role': 'admin',
     });
   }
   ```
   This is a simple Firestore update. No state emission needed — the promoted user's session will pick up the role change on their next auth state refresh. The admin's UI will re-query the users list to show updated badge.
