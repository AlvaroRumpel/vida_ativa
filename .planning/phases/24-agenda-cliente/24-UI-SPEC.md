## 4. Spacing Scale

Uses Flutter's 8-point grid expressed as `EdgeInsets` and `SizedBox` values.

| Token | Value | Usage in Phase 24 |
|-------|-------|-------------------|
| xs | 4 px | Gap between day-strip abbrev and number |
| sm | 8 px | Vertical padding inside header row; gap between header and WeekHeader |
| md | 16 px | Horizontal screen padding (all three components) |
| lg | 24 px | SafeArea top clearance before header |
| xl | 32 px | (reserved — not used in this phase) |

Exceptions: Wordmark pill uses `EdgeInsets.symmetric(horizontal: 12, vertical: 4)` —
this produces a 12px horizontal bleed past text, which is smaller than md (16px) by design.
The 4px vertical gives the pill its sport-compact feel.

## 5. Typography

Four roles used in Phase 24.

| Role | Helper | Size | Weight | Color (default) | Height |
|------|--------|------|--------|-----------------|--------|
| Display — time | `AppTheme.display(size: 42)` | 42 px | 400 (Anton) | `AppTheme.ink` | 0.92 |
| Display — day number | `AppTheme.display(size: 22)` | 22 px | 400 (Anton) | `AppTheme.ink` (overridden per state) | 0.92 |
| Display — wordmark | `AppTheme.display(size: 18)` | 18 px | 400 (Anton) | `AppTheme.paper` | 0.92 |
| Mono — all labels / abbrevs / price / date eyebrow | `AppTheme.mono(size: 11)` | 11 px | 700 (JetBrains Mono) | varies by role (see below) | default |

Rules:
- Never pass a raw `TextStyle` with `fontFamily: 'Anton'`. Use `AppTheme.display()` always.
- Never wrap Anton `Text` in a `SizedBox` with fixed height. Anton at `height: 0.92` means
  logical box height = `fontSize * 0.92`. A SizedBox shorter than that clips the cap height.
- Mono uppercase is achieved automatically by `AppTheme.mono()` via `letterSpacing` — do not
  call `.toUpperCase()` on strings; that is redundant and breaks locale-sensitive chars (e.g. "Sáb").
  Instead pass uppercase literal strings where needed (day abbrevs: "SEG", "TER", etc.).

## 6. Component Specs

### 6.1 Wordmark Header (SCHED-06)

**What it replaces:** The `AppBar` in `ScheduleScreen`.

**Widget structure:**

```
Scaffold(
  body: Column([
    SafeArea(
      bottom: false,
      child: Padding(
        padding: EdgeInsets.symmetric(horizontal: 16, vertical: 8),
        child: Row([
          // LEFT: wordmark pill
          Container(
            padding: EdgeInsets.symmetric(horizontal: 12, vertical: 4),
            decoration: BoxDecoration(
              color: AppTheme.orange,
              borderRadius: BorderRadius.circular(20),   // pill shape — full stadium
            ),
            child: Text('VIDA ATIVA', style: AppTheme.display(size: 18, color: AppTheme.paper)),
          ),
          Spacer(),
          // RIGHT: selected-day eyebrow
          Text(_eyebrowDate(_selectedDay), style: AppTheme.mono(size: 11, color: AppTheme.ink)),
        ]),
      ),
    ),
    WeekHeader(...),          // unchanged — no visual changes
    SportDayStrip(...),       // see 6.2
    SizedBox(height: 8),
    Expanded(child: SlotDayView(...)),
  ]),
)
```

**Eyebrow date format:**

```dart
String _eyebrowDate(DateTime day) {
  // Output example: "SEG, 26 MAI"
  const abbrevDays = ['SEG', 'TER', 'QUA', 'QUI', 'SEX', 'SÁB', 'DOM'];
  const abbrevMonths = ['JAN','FEV','MAR','ABR','MAI','JUN','JUL','AGO','SET','OUT','NOV','DEZ'];
  final dayName = abbrevDays[day.weekday - 1];
  final monthName = abbrevMonths[day.month - 1];
  return '$dayName, ${day.day} $monthName';
}
```

The eyebrow updates reactively: whenever `_onDaySelected(DateTime day)` calls `setState`,
`_selectedDay` changes and the `Text` widget rebuilds.

**Visual specification:**

```
┌────────────────────────────────────────────────┐
│ ████████████████████         SEG, 26 MAI       │
│ █ VIDA ATIVA    █                               │  ← 8px vertical pad above + below Row
│ ████████████████████                            │
└────────────────────────────────────────────────┘
  ↑ orange pill (#FF4D17)       ↑ JetBrains Mono 12px, ink
  ↑ Anton 18px white             letterSpacing auto
  ↑ border-radius 20px
  ↑ hpadding 12px, vpadding 4px
```

**Constraints:**
- `AppBar` parameter is removed entirely from `Scaffold`. No `appBar:` key.
- SafeArea wraps only the header row — `SlotList` scrolls beneath it naturally.
- `SafeArea(bottom: false)` — only top inset needed.
- Do not add a `Divider` or line between header and `WeekHeader`.


**Per-column layout:**

```
┌──────┐
│ SEG  │  ← AppTheme.mono(size: 11, color: AppTheme.concrete)
│  26  │  ← AppTheme.display(size: 22, color: <state-driven>)
│ ▔▔▔  │  ← 2px orange underline — only when isSelected
└──────┘
```

**Column structure per day:**

```dart
GestureDetector(
  onTap: () => onDaySelected(day),
  child: SizedBox(
    width: 40,                     // fixed column width — 7 cols × 40px = 280px < screen
    child: Column(
      mainAxisSize: MainAxisSize.min,
      crossAxisAlignment: CrossAxisAlignment.center,
      children: [
        Text(_abbrev[i], style: AppTheme.mono(size: 11, color: AppTheme.concrete)),
        SizedBox(height: 4),
        Text('${day.day}', style: AppTheme.display(size: 22, color: _numberColor(isSelected, isToday))),
        SizedBox(height: 4),
        // Underline — visible only when isSelected
        AnimatedContainer(
          duration: Duration(milliseconds: 150),
          height: 2,
          width: isSelected ? 24 : 0,      // collapses when not selected
          color: AppTheme.orange,
        ),
      ],
    ),
  ),
)
```

**Number color rules:**

```dart
Color _numberColor(bool isSelected, bool isToday) {
  if (isSelected) return AppTheme.orange;   // selected: orange number + orange underline
  if (isToday)    return AppTheme.orange;   // today (not selected): orange number, no underline
  return AppTheme.ink;                      // other days: ink
}
```

Note: When `isSelected && isToday`, the day is both — orange number + orange underline. This is
identical to the `isSelected` branch because the underline alone distinguishes selection from "today".

**Strip container:**

```dart
Padding(
  padding: EdgeInsets.symmetric(horizontal: 16),
  child: Row(
    mainAxisAlignment: MainAxisAlignment.spaceBetween,
    children: List.generate(7, (i) => _buildColumn(i)),
  ),
)
```

Use `MainAxisAlignment.spaceBetween` — 7 columns fill the horizontal extent evenly without
a `SingleChildScrollView`. No scrolling — 7 columns at 40px wide fit in 280px, well within
the 360px minimum mobile width.

**Do not use:**
- `ChoiceChip` — removed entirely.
- `SingleChildScrollView` — no horizontal scroll needed.
- Filled chip backgrounds, `selectedColor`, `backgroundColor` from chip theme.
- A dot indicator below the number (the old DayChipRow had a 5px dot — omit).

**Row structure — WITH stripe (myBooking only):**

The orange left stripe requires that the stripe fills the full row height, which is variable
(depends on Anton line height at 42px). Use `IntrinsicHeight` here — it is acceptable because
this pattern appears at most once per day (a user can have at most one booking per day).
Do NOT use `IntrinsicHeight` inside `ListView.builder` for the non-stripe rows.


**Helper values:**

```dart
double _opacity(SlotStatus status) => switch (status) {
  SlotStatus.available  => 1.0,
  SlotStatus.myBooking  => 1.0,
  SlotStatus.booked     => 0.45,
  SlotStatus.blocked    => 0.45,
};

String _statusLabel(SlotViewModel vm) => switch (vm.status) {
  SlotStatus.available  => 'DISPONÍVEL',
  SlotStatus.myBooking  => 'MINHA RESERVA',
  SlotStatus.booked     => (vm.bookerName ?? 'OCUPADO').toUpperCase(),
  SlotStatus.blocked    => 'BLOQUEADO',
};

Color _statusLabelColor(SlotStatus status) => switch (status) {
  SlotStatus.available  => AppTheme.court,      // green — positive affordance
  SlotStatus.myBooking  => AppTheme.concrete,
  SlotStatus.booked     => AppTheme.concrete,
  SlotStatus.blocked    => AppTheme.concrete,
};

String _formatPrice(double price) =>
    NumberFormat.currency(locale: 'pt_BR', symbol: 'R\$').format(price);
```

## 8. Layout Constraints

### 8.1 Screen-level column

```
Scaffold.body = Column([
  SafeArea(bottom: false) → header Row     height: auto (~52px typical)
  WeekHeader                               height: 44px (unchanged)
  SportDayStrip                            height: auto (~56px: abbrev + number + underline + padding)
  SizedBox(height: 8)
  Expanded → SlotDayView                   fills remaining viewport
])
```

No `SizedBox` between header and WeekHeader — the 8px vertical padding inside the header Row
provides sufficient separation.

### 8.2 SportDayStrip width

7 columns × 40px = 280px. Minimum supported width is 320px (small Android). The strip fits
without scroll. Do not add `SingleChildScrollView`.

### 8.3 SlotHairlineRow height

Row height is driven by Anton 42px at `height: 0.92` + vertical padding 12px×2 = logical ~63px.
Do not constrain this with a fixed `SizedBox(height: ...)`. Let content set its own height.

### 8.4 Status label column width

Fixed `SizedBox(width: 96)` for the status label. This prevents horizontal layout shift when
switching between days where different statuses appear. "MINHA RESERVA" is the widest label
at JetBrains Mono 11px — at typical DPR it is ~88px. The 96px container gives 8px safety margin.

## 10. Do Not

These anti-patterns are explicitly forbidden for Phase 24 implementation:

| Forbidden | Reason |
|-----------|--------|
| `Card(...)` widget in slot rows | Eliminates hairline layout; adds elevation and border-radius |
| `ChoiceChip(...)` in day strip | Replaced entirely by underline pattern |
| `AppBar(...)` in `ScheduleScreen` | Removed; header is inline in body |
| `Color(0xFF...)` raw hex literals | Must use `AppTheme.*` tokens |
| `SizedBox(height: H)` wrapping Anton `Text` where H < fontSize × 0.92 | Anton height clip (Pitfall 8) |
| `SingleChildScrollView` wrapping day strip | 7 × 40px = 280px — fits without scroll |
| `IntrinsicHeight` inside `ListView.builder` for non-stripe rows | Performance cost (Pitfall 5) |
| `TextStyle(fontFamily: 'Anton', ...)` constructed manually | Use `AppTheme.display()` always |
| `AppTheme.primaryGreen` or `AppTheme.brandAmber` (legacy aliases) | Use `AppTheme.court` and `AppTheme.orange` explicitly |
| `GoogleFonts.anton(...)` called directly in widgets | Use `AppTheme.display()` for the `height: 0.92` fix |
| Hardcoded padding in `ListView.builder` (horizontal: 16) | Row padding is now inside each `SlotHairlineRow` |
| Changing `ScheduleScreen` to `StatelessWidget` | Local `_weekStart` / `_selectedDay` state must stay in `State` (Pitfall 4) |
| Forgetting to call both `setState` AND `cubit.selectDay()` in week navigation | State desync (Pitfall 4) |
