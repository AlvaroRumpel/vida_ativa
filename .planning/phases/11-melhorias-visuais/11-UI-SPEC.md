## Spacing Scale

Declared values (multiples of 4). New file `lib/core/theme/app_spacing.dart` introduces these as static constants.

| Token | Value | Dart Constant | Usage |
|-------|-------|---------------|-------|
| xs | 4px | `AppSpacing.xs` | Icon gaps, status badge internal padding, color strip width on SlotCard |
| sm | 8px | `AppSpacing.sm` | Compact list item vertical padding, inline element gaps |
| md | 16px | `AppSpacing.md` | Default horizontal padding on list screens, form field spacing |
| lg | 24px | `AppSpacing.lg` | Section padding, form horizontal padding, empty-state text padding |
| xl | 32px | `AppSpacing.xl` | Layout gaps between major sections |

Source: CONTEXT.md `## UI-03: Spacing sistemático` — tokens `xs=4, sm=8, md=16, lg=24, xl=32`. RESEARCH.md confirms codebase uses these exact literal values already.

Exceptions:
- Timeline tile height is controlled by `DayView.heightPerMinute: 1.0` (60px per hour) — not an AppSpacing token. This is a `calendar_view` layout parameter, not a padding/margin value.
- Color strip on SlotCard left edge: fixed at 4px (maps to `AppSpacing.xs`) — already matches the scale.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | General text, slot start time label in timeline tile |
| Label | 14px | 400 (regular) | 1.4 | Price display, secondary metadata |
| Heading | 20px | 600 (semibold) | 1.2 | Screen app bar titles, section headings |
| Caption | 13px | 400 (regular) | 1.3 | Status badge text ("Disponível", "Minha reserva", "Bloqueado") inside timeline tiles |

Font weights in use: regular (400) and semibold (600). Two weights only.

Source: `slot_card.dart` — startTime at 16px w600 (primary label); price at 14px, bookerName at 13px use default weight (w400 per Flutter default when no weight specified); "Disponível" and "Bloqueado" labels carry no explicit weight (w400). Heading 20px is Material 3 titleMedium default confirmed in `app_theme.dart`. Medium (500) weight declared in original slot_card.dart for price/bookerName has been consolidated to regular (400) — one step on the weight scale, minimal visual impact.

## Color

| Role | Value | Dart Reference | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | `#FFFFFF` (white) | `Colors.white` / Material scaffold background | Screen backgrounds, DayView background, card surfaces |
| Secondary (30%) | `#F5F5F5` (grey 100) | `Colors.grey[100]` / `Colors.grey.shade200` | Cards, status badge container (myBooking badge), inactive list tiles, timeline empty-hour rows |
| Accent (10%) | `#2E7D32` (Material Green 800) | `AppTheme.primaryGreen` | Available slot color strip, available tile background (with 0.2 opacity), "Disponível" label text, BottomNav selected item, FilledButton background |
| Destructive | `#E53935` (Material Red 600) | `const Color(0xFFE53935)` | Blocked slot color strip, blocked tile background (with 0.2 opacity), "Bloqueado" label text |

Source: `app_theme.dart` — `primaryGreen = Color(0xFF2E7D32)`. `slot_card.dart` line 65 — `Color(0xFFE53935)` for blocked. RESEARCH.md color mapping table.

Accent reserved for:
1. Available slot indicator strip (left edge of SlotCard, 4px wide)
2. Available slot tile background fill in DayView (`primaryGreen.withOpacity(0.2)`)
3. "Disponível" status text label
4. myBooking slot tile background fill (solid `primaryGreen`)
5. BottomNavigationBar selected item indicator
6. FilledButton primary action background (book slot CTA)

Occupied slot (booked by another user) uses `Colors.grey.shade200` background — not accent, deliberately neutral.

## Interaction Contract

### Timeline Navigation (UI-02)

| Interaction | Behavior |
|-------------|----------|
| Day chip tap | Selects day, DayView updates events, scroll animates to first slot of that day |
| Available tile tap | Opens `BookingConfirmationSheet` — identical to existing flow |
| Occupied/myBooking/blocked tile tap | No action — tile is inert |
| Timeline scroll | Vertical scroll within DayView; hour labels fixed on left column |
| Day swipe | DISABLED — no horizontal swipe; navigation via DayChipRow only |

### Loading State

Shimmer effect: 3 placeholder blocks positioned at hours 09:00, 10:00, 11:00 within the DayView grid. Each block uses opacity animation (pulse 0.3–0.8 opacity, 800ms duration, repeat). Uses existing `AnimationController` pattern from `SlotSkeleton` — no shimmer package.

### DayView Configuration

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| `heightPerMinute` | `1.0` | 60px per hour — sufficient for tile content on 375px viewport |
| `backgroundColor` | `Colors.white` | Matches dominant surface color |
| `dayTitleBuilder` | `(_) => const SizedBox.shrink()` | Hides DayView's built-in header; WeekHeader+DayChipRow remain above |
| `timeStringBuilder` | `HH:mm` format (zero-padded) | Consistent with existing `slot.startTime` format in codebase |
| `showLiveTimeLineInAllDays` | `true` | "Now" indicator — native support in calendar_view, zero extra work |
| `keepScrollOffset` | `false` (default) | Auto-scroll to first slot on day change |
| `startHour` | Dynamic: `max(0, firstSlotHour - 1)` | 1h before earliest slot, clamped to 0 |
| `endHour` | Dynamic: `min(24, lastSlotEndHour + 1)` | 1h after latest slot end, clamped to 24 |
| Empty hour rows | Visible with subtle `Divider` | All hours in range shown even without slots |
