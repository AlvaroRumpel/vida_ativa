**admin_slot_state.dart:**
```dart
sealed class AdminSlotState extends Equatable { const AdminSlotState(); }
class AdminSlotInitial extends AdminSlotState { const AdminSlotInitial(); @override List<Object?> get props => []; }
class AdminSlotLoaded extends AdminSlotState {
  final List<SlotModel> slots;
  const AdminSlotLoaded(this.slots);
  @override List<Object?> get props => [slots];
}
class AdminSlotError extends AdminSlotState {
  final String message;
  const AdminSlotError(this.message);
  @override List<Object?> get props => [message];
}
```

**admin_slot_cubit.dart:**
- Constructor takes `FirebaseFirestore firestore`. Starts stream in constructor.
- `_startStream()`: streams ALL slots from `collection('slots').snapshots()` (no `isActive` filter — admin sees all). Maps with `SlotModel.fromFirestore`. Sorts by `dayOfWeek` then `startTime` (using `.compareTo()`). Emits `AdminSlotLoaded(slots)`.
- `createSlot({required int dayOfWeek, required String startTime, required double price})`: uses `_firestore.collection('slots').add({'dayOfWeek': dayOfWeek, 'startTime': startTime, 'price': price, 'isActive': true})`. Note: `.add()` is correct for slots (NOT deterministic ID — that rule only applies to BookingModel).
- `updateSlot(String slotId, {required int dayOfWeek, required String startTime, required double price})`: uses `_firestore.collection('slots').doc(slotId).update(...)`.
- `setSlotActive(String slotId, bool isActive)`: uses `_firestore.collection('slots').doc(slotId).update({'isActive': isActive})`.
- Override `close()` to cancel `_sub`.

**admin_booking_cubit.dart:**
- Constructor takes `FirebaseFirestore firestore` and `String adminUid`. Store both. Emit `AdminBookingInitial()`. Call `_loadConfig()` then `selectDate(DateTime.now())`.
- `_loadConfig()`: reads `_firestore.collection('config').doc('booking').get()`. Stores `_confirmationMode = configSnap.data()?['confirmationMode'] ?? 'manual'` in an instance field.
- `selectDate(DateTime date)`: cancels previous `_sub`. Formats date to `'YYYY-MM-DD'` string. Starts new stream: `_firestore.collection('bookings').where('date', isEqualTo: dateString).snapshots()`. Maps docs with `BookingModel.fromFirestore`. Sorts locally by `startTime ?? ''`. Emits `AdminBookingLoaded(bookings, selectedDate: date, confirmationMode: _confirmationMode)`. NO `.orderBy()` — sort locally to avoid composite index (per STATE.md decision).
- `confirmBooking(String bookingId)`: `_firestore.collection('bookings').doc(bookingId).update({'status': 'confirmed'})`.
- `rejectBooking(String bookingId)`: `_firestore.collection('bookings').doc(bookingId).update({'status': 'rejected'})`.
- `setConfirmationMode(String mode)`: `_firestore.collection('config').doc('booking').set({'confirmationMode': mode}, SetOptions(merge: true))`. Update `_confirmationMode = mode`. Re-emit current state with updated mode.
- Override `close()` to cancel `_sub`.
- Date formatting helper: `String _toDateString(DateTime d) => '${d.year}-${d.month.toString().padLeft(2, '0')}-${d.day.toString().padLeft(2, '0')}';`

CRITICAL patterns to follow:
- Use `(data['price'] as num).toDouble()` for price fields (Firestore may return int or double).
- All state classes use Equatable with const constructors.
- No `.orderBy()` on Firestore queries — sort locally in Dart.
- Cancel StreamSubscription in `close()`.
