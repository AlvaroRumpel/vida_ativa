## Interaction Contract

### Recurrence Toggle Activation

1. User sees "Reservar semanalmente" toggle (switch or checkbox) in BookingConfirmationSheet
2. Toggle is initially OFF
3. On toggle ON:
   - RecurrenceSection appears via `AnimatedSize` (250ms, easeInOut)
   - Sheet height expands
   - Focus/scroll does NOT jump to section automatically
4. On toggle OFF:
   - Section collapses (reverse animation)
   - Chip selection and slider state are RETAINED in memory (not cleared)
   - Next toggle-ON restores prior state

**Animation curve:** `Curves.easeInOut`, duration 250ms

**Rationale:** User may toggle on/off to compare single vs batch; preserving state reduces friction. Pre-populated from: CONTEXT.md §Animação de expansão.

### Availability Preview Refresh

1. User drags slider to adjust week count:
   - `onChanged`: Update label only (local DateTime arithmetic)
   - `onChangeEnd`: Trigger Firestore batch get for all candidate dates
2. Preview list updates:
   - Loading state: Gray "Verificando disponibilidade..." text
   - Results: Icon + date + optional status text
   - Scrollable if >6 items, truncated with "+ N datas" link (non-interactive, just indicator)
3. User toggles day chip:
   - Same flow: update list, trigger Firestore check on toggle release (debounce: 300ms)

**Firestore query:** Parallel `Future.wait()` for all (date, startTime) combinations. No composite index required for slot lookup (single-field equality filters on date + startTime supported). See RESEARCH.md Pattern 1.

**Error handling:** If Firestore query fails, show error banner in preview area: "Não foi possível verificar disponibilidade. Tente novamente."

**Rationale:** User sees availability in real-time as they configure. No submitting with "unknown" slots. Pre-populated from: CONTEXT.md §Preview de Datas + RESEARCH.md §Pattern 1.

### Batch Submission

1. User taps "Reservar [N] semanas/reservas" button
2. Submit state:
   - Button disabled, shows spinner inline (existing pattern from BookingConfirmationSheet)
   - RecurrenceSection grayed out (optional, for visual feedback)
3. Firestore operations:
   - `Future.wait()` executes all bookings in parallel (eagerError: false via per-element try/catch)
   - Each booking: existing `bookSlot` transaction pattern + `recurrenceGroupId` parameter
   - Race condition: If another user books a slot between preview and submit, that booking is marked as "conflicted" in result
4. Result sheet:
   - If all succeeded: Show "N reservas criadas", button "Fechar"
   - If partial: Show success count + conflict list, button "Fechar"
   - Close button pops both sheets

**Rationale:** Partial success is acceptable; user sees which slots failed and why. Pre-populated from: RESEARCH.md Pattern 3 + CONTEXT.md §Sheet de Resultado.

### Group Cancellation

1. User taps "Cancelar" in ClientBookingDetailSheet for a recurrent booking (`recurrenceGroupId != null`)
2. Dialog appears: "Cancelar reserva recorrente?" with two options
3. User selects:
   - "Cancelar só esta": Execute existing `cancelBooking(bookingId)` logic
   - "Cancelar esta e as próximas": Execute new `cancelGroupFuture(recurrenceGroupId, fromDateInclusive)` logic
4. On success:
   - Sheet closes, snack: "Reserva(s) cancelada(s)"
   - MyBookingsScreen rebuilds via stream (existing reactive pattern)
5. On error:
   - Show error message in sheet (existing pattern)

**Rationale:** Group cancellation via Firestore query + batch update (WriteBatch). Requires composite index on `(recurrenceGroupId, date)`. Pre-populated from: RESEARCH.md Pattern 4.

## Firestore Indexes Required

### Composite Index for Group Cancellation

**Collection:** `bookings`
**Fields:**
1. `recurrenceGroupId` (ASCENDING)
2. `date` (ASCENDING)

**Reason:** `cancelGroupFuture()` query has equality filter on `recurrenceGroupId` + inequality filter on `date >= today`. Requires explicit composite index. See RESEARCH.md Pitfall 2.

**Deployment:** Add to `firestore.indexes.json` before deploy. Firebase console will auto-create if omitted, but with runtime error on first query attempt in production.
