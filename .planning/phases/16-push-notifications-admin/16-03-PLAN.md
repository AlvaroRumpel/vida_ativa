Deploy both the Cloud Functions and the hosting (web app) to the staging Firebase project in a single command.

First, build the Flutter web app with staging config and VAPID public key:
```bash
cd F:/_geral/Projetos/vida_ativa
flutter build web --dart-define=ENV=staging --dart-define=VAPID_PUBLIC_KEY=<VAPID_PUBLIC_KEY>
```

The VAPID public key is obtained from the Firebase Console:
- Go to Firebase Console → Project settings → Cloud Messaging → Web configuration
- The "Key pair" section shows the VAPID key pair — copy the public key

Then deploy using the staging alias from `.firebaserc`:
```bash
firebase use staging
firebase deploy --only functions,hosting
```

If `firebase.json` is missing a `"functions"` section (check first), add it before deploying:
```json
"functions": {
  "source": "functions",
  "runtime": "nodejs20"
}
```

IMPORTANT: If the Cloud Function deployment fails with "Node.js 20 not available", try `"runtime": "nodejs18"` in `firebase.json` functions section.

After deploy, verify the function appears in Firebase Console → Functions.

**Background notification test:**
5. Close Browser A (or navigate away from the admin tab)

7. Verify Browser A (or OS notification area) receives a notification titled "Nova Reserva"
8. Verify notification body contains the client's display name and booking time (e.g., "João Silva — 09:00")

**Foreground notification test (bonus):**
9. Open Browser A as admin (keep tab open in admin panel)
10. In Browser B, create another booking
11. Verify a SnackBar appears in the admin panel with the notification title and body

**Service worker check:**
12. In Browser A DevTools → Application → Service Workers → verify `firebase-messaging-sw.js` is listed as "activated and running"
