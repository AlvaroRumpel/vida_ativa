## Component Specifications

### PixPaymentScreen — Countdown Timer Addition

**Location:** `lib/features/booking/ui/pix_payment_screen.dart`

**Countdown Display:**
- **Format:** "MM:SS restantes" (minutes:seconds remaining), centered
- **Position:** Below QR code, above copy-paste code section
- **Font size:** 24px, weight 700 (bold), color primaryGreen by default
- **Color override:** Change to error red (#C62828) when **< 2 minutes remaining**
- **Implementation:** `Timer.periodic(Duration(seconds: 1), ...)` updating `setState()`
- **Start/stop:** Initiated in `initState()`, cancelled in `dispose()` (CRITICAL for memory)
- **Update frequency:** Every 1 second tick

**"Gerar novo QR" Button (Regeneration):**
- **Triggered:** When countdown reaches 00:00
- **Button type:** `FilledButton.icon` with `Icons.refresh`
- **Button text:** "Gerar novo QR"
- **Button color:** primaryGreen background, white text
- **Button size:** Full width, 48px height
- **Spacing:** 16px gap above (replacing countdown when expired)
- **Behavior:** Calls `_generateQr()` → new `createPixPayment` CF call → new PaymentRecord → countdown restarts with fresh 30min window
- **State transitions:**
  1. Countdown visible + ticking → QR enabled
  2. Countdown = 00:00 → Countdown disappears, button appears, QR gets 0.5 opacity overlay
  3. User taps "Gerar novo QR" → Loading state, new timer starts

**QR Visual When Expired:**
- **Overlay effect:** Semi-transparent grey (Colors.grey.withOpacity(0.3)) over QR image
- **Text below QR:** "QR expirado. Gere um novo acima." (14px, error color, italic)

**Copy-paste Code Section:**
- **No change** to existing layout below countdown/button

### PixPaymentScreen — Real-Time Update Listener

**Location:** `lib/features/booking/ui/pix_payment_screen.dart`

**StreamSubscription Pattern:**
- **Source:** `FirebaseFirestore.instance.collection('bookings').doc(widget.bookingId).snapshots()`
- **Initialization:** `initState()` after QR generation
- **Listener:**
  - If booking.status changes to `confirmed`:
    - Cancel countdown timer immediately
    - Show SnackBar: "Pagamento confirmado! Reserva garantida." (2s duration)
    - Navigate: `context.go('/bookings')` (via go_router, not Navigator.pop)
  - If booking.status changes to `expired`:
    - Cancel countdown timer
    - Show SnackBar: "Pagamento expirado. Gere novo QR." (3s duration)
    - UI remains on screen (user can regenerate)
- **Cleanup:** Cancel subscription in `dispose()` (CRITICAL)
- **Guard:** Check `if (!mounted) return;` before setState/navigation

### Cloud Functions UI Contract (Backend → Frontend)

**`handlePixWebhook` Response:**
- **HTTP Status:** 202 Accepted (returned immediately)
- **Body:** `{ "success": true }` (minimal, no error details exposed)
- **Processing:** Happens asynchronously after 202 sent
- **Side effect:** Booking document updated in Firestore (triggers BookingCubit stream, which auto-updates PixPaymentScreen listener)

**`expireUnpaidBookings` Response:**
- **Type:** Scheduled function, no HTTP response
- **Side effect:** Marks booking status as 'expired', which triggers stream listeners in PixPaymentScreen and MyBookingsScreen

## Interaction Patterns

### Countdown Update Loop
1. `initState()` → `_startCountdown()` initializes `Timer.periodic(Duration(seconds: 1), ...)`
2. Each tick: Calculate `_expiresAt.difference(DateTime.now())`
3. If negative: Cancel timer, set `_remaining = Duration.zero`, trigger UI rebuild
4. `dispose()` → Cancel timer (CRITICAL — prevents memory leak)

### Real-Time Payment Confirmation
1. PixPaymentScreen active, user waiting for payment
2. User pays via Mercado Pago app
3. Mercado Pago sends webhook to `handlePixWebhook` CF
4. CF verifies signature, updates booking to `status: 'confirmed'` in Firestore
5. BookingCubit stream listener detects change
6. PixPaymentScreen's `_bookingSubscription` listener fires
7. SnackBar shown: "Pagamento confirmado! Reserva garantida."
8. Navigation: `context.go('/bookings')` (go_router)
9. MyBookingsScreen displays, BookingCard shows "Confirmado" badge

### Admin Manual Fallback
1. Admin opens AdminBookingDetailSheet for `pending_payment` booking
2. Button "Confirmar pagamento manual" visible
3. Admin taps → Confirmation dialog
4. Admin confirms → `adminBookingCubit.confirmBooking()` called
5. PaymentRecord updated to `status: 'paid'` (atomic transaction)
6. Booking status → `'confirmed'`
7. AdminBookingCard badge updated to "Pix pago"
